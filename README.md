# FIRonPipelineMipsFPGA

В этом проекте реализован 32-битный конвейерный процессор MIPS с использованием SystemVerilog. упрощенную функциональную схему цифрового фильтра (рис. 9.1,а).
Непрерывный сигнал x(t) поступает на вход аналого-цифрового преобразователя (АЦП), который фиксирует значения x(n) сигнала в дискретные моменты времени t = nT, n = 0 , 1, ... и преобразует их в цифровой код в виде двоичного числа. Последовательность x(n) поступает в процессор, состоящий из арифметического устройства (АУ) и памяти (П). В процессоре осуществляется преобразование последовательности x(n) в соответствии с определенным алгоритмом. В результате на его выходе образуется последовательность y(n). Последовательность y(n) поступает на цифро-аналоговый преобразователь (ЦАП), в котором текущее значение y(n), представленное в цифровом виде, преобразуется в постоянное напряжение, удерживаемое в течение соответствующего интервала дискретности.

Добавление регистров между блоками комбинацмонной логики — это и есть конвейеризация. Конвейеризированные схемы в большинстве случаев могут работать на больших частотах, чем те, которые без конвейера, что демонстрируется на рисунках ниже, где после добавление промежуточного регистра период дескритезации изменяется с 13 нс до 7 нс, то есть уменьшается почти в 2 раза, а значит в 2 раза увеличивается частота тактирования.

Структурная схема модуля верхнего уровня состоит из модулей памяти данных и инструкций, а также устройства управления и трактом данных, представляющим собой логику путешествия данных между функциональными блоками процессора. При этом принимается, что и память команд, и память данных находятся снаружи процессора и подсоединены к нему с помощью шин адреса и данных. Это более реалистичный сценарий, потому что большинство процессоров используют внешнюю память.

<img width="974" height="826" alt="image" src="https://github.com/user-attachments/assets/87e55f9d-1235-44fb-adfa-a5f9232b0e96" />

Тракт данных процессора состоит стандартных пяти этапов конвейера: выборка инструкций (IF), декодирование инструкций (ID), выполнение (EX), обращение к памяти (MEM) и обратная запись (WB). Программная реализация имеет модульную структуру, для каждого внешнего и внутреннего компонента процессора в коде предусмотрен отдельный файл программного модуля. После произведения компиляции проекта был получен результат, который наглядно визуализируется в утилите «RTL Viewer» - означает уровень регистровых передач, способ описания электронного устройства на основе передачи данных между его регистрами.
Из рисунка ниже видно, что визуализация блок-схемы не содержит ошибки: отсутствие соединений между портами различных модулей, неверное соединение портов модулей, ошибки в ширине шинны соединения между модулями, следовательно программная реализация была составлена верно и успешно скомперирована.

Реализация поддерживает арифметические, логические операции, операции с памятью, ветвления и переходы.

| Инструкция MIPS | Соответсующая операция АЛУ | Функционал операции АЛУ |
|-----------------|----------------------------|-------------------------|
|      add        |          010 (ADD)         |        Сложение         |
|      addi       |          010 (ADD)         |        Сложение         |
|       sw        |          010 (ADD)         |        Сложение         |
|       lw        |          010 (ADD)         |        Сложение         |
|      lui        |          001 (LUI)         | Загрузка верхних 16 бит |
|      ori        |          101 (OR)          | Загрузка нижних 16 бит  |
|      srav       |          100 (SRA)         |                         |
|      slt        |          111 (SLT)         | Сравнение (если меньше )|
|      bne        |          110 (SUB)         |  Вычитание (сравнение)  |
|       j         |       Не используется      | Арифметический сдвиг вправо |

Для проведения полноценной варификации процессора необходимо провести нефункциональное моделирование, в котором уже учитываются задержки между логическими элементами и компонентами микросхемы ПЛИС. Для этого надо определить такой период тактового сигнала, который будет обеспечивать такое быстродействие схемы, чтобы компенсировать всевозможные задержки распространения сигналов через элементы, а также время на предустановку и ожидание компонентов последовательной логики. Но такой расчет осуществляется через величины задержек, соответствующих специализированным процессорным блокам. А для ПЛИС типа Cyclone V/VI — это задержки через программируемую логику (регистры, LUT, Memory blocks, DSP blocks). Расчет через эти задержки осуществляется в Quartus инструментом анализатора временных параметров (Timequest Timing Analyzer). На рисунке ниже представлен отчет из Timequest Timing Analyzer, из которого видно, что значение временного отставания прибытия данных от времени их ожидания (slack) равно -6,516 нс. Временное отставания данных представляет собой величину разницы между ограничением тактового периода (по умолчания равной 1,000 нс) и текущей (смоделированной) задержкой пути.

Таким образом, период тактового сигнала с запасом составит 8 нс.

<img width="1919" height="1080" alt="image" src="https://github.com/user-attachments/assets/f54fb296-f750-473a-b6ee-31d260d3318f" />

Реализация была проверена с помощью относительно сложной тестовой программы - казуального нерекурсивного КИХ-фильтра скользящего среднего. При этом поскольку данная реализация процессора работает только с целыми числами, а коэффициенты, сформированные при моделировании сигнала в Mathcad на рисунке ниже были с плавающей точкой, были округлен до ближайшего целого. Сам алгоритм КИХ-филььтра скользяшего среднего будет реализован через инструкции add и srav (сделано из соображения, что делить на 2 в степени менее ресурсозатратно, чем на другие числа, так как это деление реализуется сдвигом битов в регистре).
