# PipelineMips

В этом проекте реализован 32-битный конвейерный процессор MIPS с использованием SystemVerilog. 
Добавление регистров между блоками комбинацмонной логики — это и есть конвейеризация. Конвейеризированные схемы в большинстве случаев могут работать на больших частотах, чем те, которые без конвейера, что демонстрируется на рисунках ниже, где после добавление промежуточного регистра период дескритезации изменяется с 13 нс до 7 нс, то есть уменьшается почти в 2 раза, а значит в 2 раза увеличивается частота тактирования.

Структурная схема модуля верхнего уровня состоит состоящего из модулей памяти данных и инструкций, а также устройства управления и трактом данных, представляющим собой логику путешествия данных между функциональными блоками процессора. При этом принимается, что и память команд, и память данных находятся снаружи процессора и подсоединены к нему с помощью шин адреса и данных. Это более реалистичный сценарий, потому что большинство процессоров используют внешнюю память. Программная реализация имеет модульную структура, для каждого внешнего и внутреннего компонента процессора в коде предусмотрен отдельный файл программного модуля.

<img width="974" height="826" alt="image" src="https://github.com/user-attachments/assets/87e55f9d-1235-44fb-adfa-a5f9232b0e96" />

Реализация поддерживает арифметические, логические операции, операции с памятью, ветвления и переходы.
После произведения компиляции проекта был получен результат, который наглядно визуализируется в утилите «RTL Viewer».
Тракт данных процессора состоит стандартных пяти этапов конвейера: выборка инструкций (IF), декодирование инструкций (ID), выполнение (EX), обращение к памяти (MEM) и обратная запись (WB).
